# Design system — Uljas architecture gallery

The look of `index.html`, documented so it can be re-themed or extended without
reverse-engineering the CSS. Everything lives in one `<style>` block generated by
`build-gallery.py`; there are no external assets.

## Concept

**A plate register (*kuvasto*).** The 24 views are treated as numbered plates in an
engineering atlas, coded by the formal notation they use:

| Prefix | Section | Notation |
|--------|---------|----------|
| **K** | Konteksti & prosessit | ArchiMate 3.2 |
| **T** | Tietomallit | UML 2.5 (class/data) |
| **S** | Sekvenssit | UML 2.5 (sequence) |

The code (`K1`, `T11`, `S3`) is the anchor: it appears large in the header and leads
every register row. This is the one **structural device** — the letter encodes real
information (which notation you're looking at), not decoration.

The rest is deliberately quiet: warm paper, ink, hairlines, and a **single indigo
accent** used only for the active plate, focus rings, and accent text. The diagram
canvas ("sheet") stays light in both themes because the SVGs carry their own colours
and must stay legible — that's a functional rule, not a stylistic one.

## Colour tokens

Four `:root` blocks drive theming: light defaults, a `prefers-color-scheme: dark`
block, and explicit `[data-theme="light"|"dark"]` overrides so a manual toggle always
wins. To re-theme, edit these values — nothing else references raw hex.

| Token | Role | Light | Dark |
|-------|------|-------|------|
| `--paper` | app ground | `#f1f0ea` | `#131419` |
| `--panel` | elevated surface (header, footer, active row) | `#ffffff` | `#1b1d25` |
| `--rail` | register sidebar | `#f7f6f1` | `#171922` |
| `--ink` | primary text | `#1b1a16` | `#e9e7df` |
| `--muted` | secondary text (questions, section titles) | `#6b675e` | `#9c988e` |
| `--faint` | metadata (counts, ids, path) | `#9b978b` | `#6f6c63` |
| `--line` | hairline dividers | `#ddd9cf` | `#282b34` |
| `--line-2` | control borders | `#c9c4b6` | `#3a3e49` |
| `--accent` | active marker, focus | `#4b45c4` | `#9b96f6` |
| `--accent-ink` | accent text (active code, kind chip) | `#3a34a6` | `#b9b5fb` |
| `--wash` | accent tint (kind chip bg) | `#ecebfb` | `#22233a` |
| `--sheet` | diagram canvas — always light | `#ffffff` | `#ffffff` |
| `--sheet-line` | sheet border | `#e7e3d8` | `#d8d4cb` |
| `--ok` / `--ok-wash` | "asettelu ok" chip | `#2f6b45` / `#e6f1ea` | `#7ad3a0` / `#123123` |
| `--warn` / `--warn-wash` | "tiheä asettelu" chip | `#8a5a12` / `#f6ead0` | `#e7c17c` / `#332913` |
| `--shadow` | elevation | soft, warm | soft, black |

Hover states are derived, not stored: `color-mix(in srgb, var(--accent) 7–10%, transparent)`.

## Typography

Two families, no web fonts (keeps the file self-contained and offline-safe):

- `--sans` → `system-ui, -apple-system, "Segoe UI", Roboto, Helvetica, Arial, sans-serif`
  — view/plate titles, register labels, questions.
- `--mono` → `ui-monospace, "SF Mono", SFMono-Regular, "JetBrains Mono", Menlo, Consolas, monospace`

**The signature is mono-forward chrome.** Everything structural is monospace — eyebrows,
section headers, plate codes, ids, counts, chips, the file path — so the technical
register reads as an instrument, while titles in sans carry the content.

| Element | Family | Size / weight / tracking |
|---------|--------|--------------------------|
| Plate code (header) | mono | 27px · 600 · tabular |
| Masthead title | sans | 20px · 640 · −0.02em |
| View title | sans | 19px · 640 · −0.02em |
| Question | sans | 13px · `--muted` |
| Register label | sans | 12.5px (600 when active) |
| Register code | mono | 12px · 600 |
| Section header | mono | 10.5px · uppercase · 0.15em · `--muted` |
| Chip | mono | 10.5px · 600 · 0.03em |
| Eyebrow | mono | 11px · uppercase · 0.16em · `--accent-ink` |
| Register meta / footer | mono | 9.5–11px · `--faint` |

## Layout

```
#app  ── CSS grid, 304px rail + 1fr stage, height 100vh, overflow hidden
├─ #rail (aside)              ├─ #stage (main, flex column)
│  ├─ .mast     border-bottom │  ├─ #head    border-bottom, --panel
│  ├─ #register (scrolls)     │  ├─ #viewer  flex:1, scrolls, centers #sheet
│  └─ .rail-foot border-top   │  └─ #foot    border-top, --panel
```

- The rail and the stage's header/footer sit on `--panel`/`--rail`; only the scrolling
  areas (`#register`, `#viewer`) show `--paper`.
- `#sheet` is a rounded, bordered, shadowed card on `--sheet`; the SVG fills its width
  (`svg{width:100%;height:auto}`) and the model's own visible-title text is hidden
  (`[data-arch-a11y="visible-title"]{display:none}`) since the chrome shows it.

## Components

- **Register row** — `grid: 34px code · 1fr label · auto meta`. Active row lifts onto
  `--panel` with a 2px `--accent` left border and `--shadow`; the code turns `--accent-ink`.
- **Chips** — mono pills: `chip-kind` (accent wash), `chip-ok`, `chip-warn`,
  `chip-count` (outline only). One notation/status/size read at a glance.
- **Zoom group** — segmented `– · Sovita · +`, mono, hairline-divided; range 25–400%,
  "Sovita" resets to 100% (fit-to-width).
- **Icon button** — theme toggle (`◐`), 35×34, `--line-2` border.

## Motion

One gesture: on plate change, `#sheet` replays a 0.18s fade + 5px rise (`@keyframes swap`).
Fully suppressed under `prefers-reduced-motion: reduce`. Nothing else animates.

## Accessibility

- `:focus-visible` → 2px `--accent` outline, 2px offset, on every control.
- Register is `role="tablist"`; rows are `role="tab"` with `aria-selected`; **↑/↓** move
  between plates and keep focus.
- SVGs keep `role="img"` + `<title>`/`<desc>`; the redundant visible title is hidden
  visually but the accessible name remains.

## Responsive

At `max-width: 900px` the grid collapses to a single column: the rail becomes a
top bar (`max-height: 40vh`) and the header plate code shrinks to 22px.

## Extending

- **Re-theme:** edit the four `:root` blocks. Every surface is a token.
- **Add a view:** render it into the dediren package and run `python3 docs/build-gallery.py`
  — grouping and the `K/T/S` code come from `group_of()`, counts from `render-metadata`.
- **Density rule:** a view is flagged "tiheä" at `DENSE_EDGES = 50` relationships
  (`build-gallery.py`) — tune there, not in the markup.
